# ALLOCATION_LOGIC

## 設計の根拠
学習科学の知見をベースに配分ロジックを設計する。
参照研究: Cepeda et al. 2006/2008, Rohrer & Taylor 2007, Kornell & Bjork 2008, Bjork 1994, Roediger & Karpicke 2006, Ariga & Lleras 2011

---

## ステップ1：科目ごとの重みを計算

```
gap        = max(target_score - previous_score, 5)   // 最低5（目標達成済みも0にしない）
difficulty = 1 + (previous_score / 100)              // 高得点帯ほど伸ばすのが難しい
efficiency = 1 / (1 + log(1 + study_minutes / 30))  // 過去の勉強量が少ない科目ほど伸びしろ大
weight     = gap × difficulty × efficiency
```

### null値の扱い
- `previous_score` が null（新科目）→ 50 として補完
- `previous_study_minutes` が null → 0 として補完

---

## ステップ2：配分を計算

```
raw_share    = weight / Σweight
capped_share = min(raw_share, 0.40)   // 1科目に40%以上は割り当てない
planned_min  = total_minutes × normalize(capped_share)
```

### 10分刻みへの丸め
```
rounded = round(planned_min / 10) * 10
```
- 丸めて0分になる科目は強制的に10分にする
- 丸め後の端数（total - Σrounded）は最大weight科目に加算

---

## ステップ3：エッジケース処理

| ケース | 対処 |
|---|---|
| 全科目が目標達成済み | 全weight = 5（floor）→ 均等配分 |
| Σweight = 0（ゼロ除算） | `total_minutes / 科目数` で均等配分にフォールバック |
| 目標を下げた科目（previous > target） | weight = 5（floor）。UIで「維持学習として含めています」表示 |
| 勉強日が1日のみ | 日分配ループで `days.length === 1` を明示ハンドリング |
| 丸め後の合計がずれる | 最大weight科目に端数を加算して吸収 |

## ステップ4：日ごとの配置

### 4-1. 重い科目を前半に配置
根拠: Bjork 1994「認知負荷が高い時間帯に難しい素材を取り組む方が定着率が高い」

```
heavy_subjects = weight上位50%の科目
前半40%の日（例：10日中1〜4日目）に、heavy科目の65%を配置
残り35%を後半に配置
```

- 1科目1日あたりの上限: **90分**（超えた分は翌日以降に溢れさせる）
- UIで理由を表示: 「苦手科目を前半に配置しています。集中力が高いうちに取り組もう！」

### 4-2. インターリービング（科目を1日に混ぜる）
根拠: Rohrer & Taylor 2007「1科目まとめより複数科目を交互にやる方がテスト成績が43%高い」

```
1日に割り当てる科目は2〜3科目を混ぜる（1科目のみの日を避ける）
各科目のブロックは25〜30分単位を推奨
```

- UIで理由を表示: 「複数科目を交互に取り組むと記憶に残りやすいです」

### 4-3. 同じ科目を連続日に固めない
根拠: Cepeda et al. 2008「分散学習は詰め込みより長期記憶に有利」

- 同じ科目が3日連続しないように配置する
- 最低3セッションを別々の日に分散させる

---

## ステップ5：テスト期間中の切り替え

根拠: テスト日自体はもともとの勉強日として計算しない（ボーナス日として扱う）

```
テスト1日目の夜 → 翌日（テスト2日目）の科目のみ割り当て
テスト2日目の夜 → 翌日（テスト3日目）の科目のみ割り当て
テスト最終日    → 割り当てなし
```

### 複数科目がある夜の配分
- 翌日の科目が2科目なら 6:4（weight多い方を多く）

### 自動適用 + バナー表示
- 「テスト日程に合わせてプランを調整しました」をバナーで通知
- 中間日は「他の科目も追加する」逃げ道を用意
- 最終日は上書き不可。「お疲れ様！今日は休んでください」を表示

### 1日のみのテスト
- 切り替えロジック不要。テスト前日までで通常通り計算

---

## 参考：学習セッション長の目安

根拠: Ariga & Lleras 2011, Harris Cooper（OECD PIASデータ）

| 対象 | 推奨セッション長 | 休憩 | 1日の上限 |
|---|---|---|---|
| 中学生 | 25〜30分 | 5分 | 2時間 |
| 高校生 | 45〜50分 | 10分 | 3.5時間 |

- 上限を超えると疲労による「浅い処理」になり定着しない
- UIでは1日の割り当て合計が上限に近づいたら目安として表示する
