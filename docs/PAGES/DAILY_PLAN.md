# DAILY_PLAN

## 目的
- 日付ごとに、何の科目をどのくらいやるかを確認または調整する

## 表示
- 上部:
  - 教科別のおすすめ配分サマリー
- 中部:
  - 日付一覧、または時間のないカレンダー
- 日付
- 科目
- 目安時間

例:
- 数学 8時間
- 9/10 数学 45分
- 9/11 英語 30分 / 理科 30分
- 9/12 数学 30分

## 自動モードの導線
- `PLAN_MODE` で設定完了後に入る
- 上部におすすめ配分サマリーが表示される
- 自動生成された日ごとのプランを確認する
- 必要なら軽く修正する
- `手動で調整した行は維持されます` を再計算導線の近くに表示する

## 手動モードの導線
- `PLAN_MODE` で `手動で組む` を押したら即入る
- 上部におすすめ配分サマリーが表示される
- 日付一覧をタップすると、その日の科目と目安時間を追加/編集できる

## 手動モード未入力時
- まだどの日にも学習プランを置いていない状態を持つ
- このときは空状態UIを出す

表示例:
- `まだ学習プランがありません`
- `日付を押して、やる科目と目安時間を置いていきましょう`
- `最初の1日を追加する`

主操作:
- 日付をタップして追加
- 既に入力済みの日付をタップして編集

## 編集ルール
- 手動上書きは行単位で行う
- 許可する操作は `科目変更` `時間変更` `行追加` `行削除`
- 手動で変更した行には `手動` バッジを付ける
- 手動で変更した行は `source=manual` として扱う
- `source=manual` の行は再計算で保持する
- 再計算時は `auto` 行のみ再生成する
- 不足または余剰の配分差分は `auto` 行で吸収する
- `manual` 行の追加は既存科目に対してのみ許可する
- `manual` 行の合計が教科の必要総時間を超えた場合は超過を警告し、`auto` 行は 0 までしか削らない
- `manual` 行を含めた結果、その日の合計が利用可能時間を超える場合は警告を表示する

## 重要な意味
- この画面で `教科ごとの総量` と `日ごとの割り当て` をまとめて可視化する
- `次へ` を押した後に、今回の全体像が一気に見えることを重視する
- `残り時間` は初回表示では出さず、進捗登録後に返す

## 画面内の一言
- 「複数科目を交互に取り組むと、記憶に残りやすいです」
  - 根拠: 1科目まとめより複数科目を交互にやる方がテスト成績が43%高い（Rohrer & Taylor 2007）
- 1日の合計が中学生2時間・高校生3.5時間を超えたら: 「少し詰め込みすぎかも。休憩も計画に入れてみよう」

## やらないこと
- 時刻の入力
- Googleカレンダーのような時間単位編集
- 複雑なドラッグ前提UI
- 日ごとの例外設定
