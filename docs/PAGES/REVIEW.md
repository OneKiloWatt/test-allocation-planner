# REVIEW

## 目的
- 今回テストの結果を、次回に使える形で振り返る

## 基本方針
- 振り返り画面は単なる一覧にしない
- `テストごとの結果カード` を基本表示にする

## カードに入れる情報
- テスト名
  - 例: `1年1学期中間`
- 実施時期
  - 学年 / 学期
- 科目ごとの勉強時間
- 科目ごとの目標点
- 科目ごとの実際の点数
- 達成 / 未達
- 一言まとめ
  - 例: `数学は達成、理科は時間不足傾向`

## 表示イメージ

```text
┌────────────────────────────┐
│ 1年1学期中間               │
│ 数学  8時間 / 目標70 / 結果72 │
│ 英語  5時間 / 目標80 / 結果76 │
│ 理科  3時間 / 目標65 / 結果58 │
│ まとめ: 数学は達成、理科は不足 │
│ [この結果を次回に引き継ぐ]    │
└────────────────────────────┘
```

## 主な操作
- `この結果を次回に引き継ぐ`
- `今回の結果を修正する`
- `次のテストを作成する`

## この画面でやらないこと
- 長期推移グラフを重く出す
- 他人比較
- 偏差値や順位の表示

## この画面が重要な理由
- ユーザーは単に過去ログを見たいのではない
- `この時はどのくらい勉強して、目標に届いたか`
- `次回は何を変えるべきか`
  を短時間で把握したい

そのため、一覧よりカードの方が価値が伝わりやすい
