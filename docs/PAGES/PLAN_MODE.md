# PLAN_MODE

## 目的
- おすすめ配分を、実際の日程へどう落とすかを決める

## 前提
- この画面の前に `目標点数入力` を終えている
- ここでは `いつ勉強できるか` と `どう落とすか` だけに集中させる

## 最初に選ばせるもの
- `自動で組む`
- `手動で組む`

## 自動で組む
### 主入力
- 部活がある平日の勉強時間
- 部活がない平日の勉強時間
- 土日の勉強時間
- 部活がある曜日

### 詳細設定
- テスト1週間前は部活なし扱い
- 勉強開始日を変更
  - デフォルトは登録日
  - `登録日` から `テスト前日` まで変更できる

### UI方針
- 主入力は最初から見せる
- `通常は今日から計画します` を主入力の近くに表示する
- `開始日: 今日 変更` のように現在値を要約表示する
- `こだわって設定する` または `詳細設定（任意）` を押すと詳細設定が開く
- 設定完了後に `日ごとの学習プラン` 画面へ進む

### MVPでやらないこと
- 例外日の細かい上書き
- 特殊ケースの複雑なパターン入力
- `この日だけ勉強不可` のような日別例外設定

### 注意書き
- `日ごとの例外設定は今は未対応です`

## 手動で組む
### できること
- 日付ごとに科目を置く
- その日の科目と時間を編集する

### やらないこと
- Googleカレンダーのような時間単位の予定編集
- ドラッグ前提の複雑なUI

### UI方針
- 最初に生活パターン入力を要求しない
- `手動で組む` を押したら、すぐ `日ごとの学習プラン` 画面へ進む
- ベースは `時間のないカレンダー` または `日付一覧`
- 主操作は `日付を押して追加/編集`
- `+` は補助としてあってもよいが、主操作にしない
- まずは日ごとの学習テーマを置けるだけでよい
